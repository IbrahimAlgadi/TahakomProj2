## Simple User Story Template

**Story Title:**  
*Short, clear summary (e.g., "Password Reset Feature")*

**User Story:**  
> As a [user type], I want [feature or action] so that [benefit].

**Acceptance Criteria:**  
- [ ] List the main things that must be true for the story to be done.

## Simple Task List

**Tasks:**  
- [ ] Task 1 (e.g., Create password reset form)
- [ ] Task 2 (e.g., Build backend to send reset email)
- [ ] Task 3 (e.g., Test the password reset process)

## Simple Subtask List (for each task)

**Example Task:** Create password reset form  
- [ ] Subtask 1 (e.g., Add "Forgot Password?" link to login page)
- [ ] Subtask 2 (e.g., Design email input field)
- [ ] Subtask 3 (e.g., Add submit button)

## Example

**Story Title:** Password Reset Feature

**User Story:**  
> As a user, I want to reset my password so that I can log in if I forget it.

**Acceptance Criteria:**  
- [ ] User can request a password reset link.
- [ ] User receives a reset email.
- [ ] User can set a new password using the link.

**Tasks:**  
- [ ] Create password reset form
- [ ] Build backend to handle reset requests
- [ ] Test the password reset process

**Subtasks for "Create password reset form":**  
- [ ] Add "Forgot Password?" link to login page
- [ ] Design email input field
- [ ] Add submit button

**Tips for Keeping It Simple:**
- Use short, action-based sentences.
- Only include what’s necessary to complete the story.
- Avoid technical jargon unless everyone understands it.

Let me know if you want this in a specific format (like a table or checklist) or for a different type of feature!
